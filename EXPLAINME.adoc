== Claim 1: Two-project monorepo for NUJ journalists and fact-checkers

[quote, README]
____
Social media analysis, monitoring, and verification tools. A monorepo
containing two major projects built for the National Union of Journalists
(NUJ) and general-purpose fact-checking workflows.
____

*How it works.*
The repo is organised as two independent sub-projects under `dipstick/` and
`polygraph/`.
Dipstick is the policy-monitoring arm: a Rust/Axum collector scrapes platform
terms-of-service pages, SHA256-checksums each version, and feeds changes to a
ReScript/Deno analyzer that classifies severity and drafts NUJ guidance.
A Deno publisher then routes the guidance to journalists via email or webhook
with a 19-layer safety guardrail and a 5-minute grace period for recall.
An Elixir/Phoenix LiveView dashboard (`services/dashboard/`) provides
real-time approval workflows.
Polygraph is the fact-checking arm: a Rust/Axum backend provides a GraphQL
API backed by ArangoDB (multi-model graph + document store) and XTDB
(bitemporal audit trail), with a ReScript frontend and browser extension for
in-context verification on social media pages.
Both sub-projects share a Dragonfly cache layer (Redis-compatible), Loki
structured logging, and Prometheus metrics.

*Honest caveat.*
Dipstick is described as 40 % complete; the collector, publisher, and
dashboard have the most runtime coverage.
The Ada/SPARK TUI (`services/tui-ada/`) and the XTDB temporal service are
architectural commitments with partial implementation — formal verification of
the SPARK interface is in progress, not complete.

== Claim 2: 19-layer safety guardrail system for journalist communications

[quote, README]
____
Publisher: Deno — Email delivery with 19-layer safety guardrails, 5-minute
grace period.
____

*How it works.*
`dipstick/services/publisher-deno/` implements the safety pipeline as a
sequence of middleware functions applied before any email or webhook dispatch:
authentication, rate-limiting, content-class filtering, duplicate suppression,
recipient allowlist enforcement, rollback locking during the 5-minute recall
window, and delivery-receipt logging.
Each layer is a Deno-native module that either passes the payload forward or
rejects it with a structured error captured in the analytics store.
`dipstick/services/gateway-rescript/` acts as the federated GraphQL entry
point stitching the Analyzer, Publisher, and Agent Swarm schemas together
into a single typed API surface for the Dashboard.

*Honest caveat.*
The 19-layer count is a design specification; the number of fully-tested
layers in `dipstick/services/publisher-deno/` should be verified against the
test suite in `dipstick/services/publisher-deno/` before treating the figure
as complete coverage.

== Dogfooded Across The Account

[cols="1,3", options="header"]
|===
| Tool / Repo | How social-media-tools uses it

| `panic-attacker`
| Pre-commit security scan across all services in the monorepo

| `contractile.just` / contractiles
| Hyperpolymath contractile integration hook at monorepo root and per-service

| `stapeln.toml`
| Container build manifest; Chainguard Wolfi base for each service image

| `verisimdb`
| Dipstick Collector writes policy-change time-series to a per-project VeriSimDB instance

| `svalinn`
| Edge gateway validating JSON Schema and policy before container deployments in the Dipstick stack

| `selur`
| `selur-compose` reads `compose.toml` to orchestrate the verified `nuj-stack` container graph

| `cerro-torre`
| Formal container builder signs SBOMs and image attestations for Chainguard-based images

| `vörðr`
| Runtime engine executes containers with verified attestation and RBAC policy

| `0-AI-MANIFEST.a2ml`
| AI manifest present at monorepo root for agent session initialisation
|===
