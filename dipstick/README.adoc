== Containerization & Orchestration

This repo now builds a **Chainguard-based container** (multi-stage) rather than relying on Podman compose. The container recipe is assembled from the ecosystem services you already maintain:

* **Svalinn** – edge gateway + `svalinn-compose` CLI (Deno/ReScript) that validates JSON Schema and policy before letting deployments run.
* **Cerro Torre** – formal container builder (Ada/Rust) that signs SBOMs, attestations, and image bindings required by the verified container spec.
* **Selur** – the orchestration CLI that replaces Podman compose (`selur-compose`), coordinates service graphs, and integrates with Svalinn/Vörðr.
* **Vörðr** – runtime engine (Rust/Idris) that executes containers with verified attestation + RBAC policy.

The new `Containerfile` (see `containerfiles/chainguard.Containerfile`) will:

. use `cgr.dev/chainguard/wolfi-base:3.20` as the final runtime so the stack ships on a Chainguard-approved base that still runs Deno/Elixir/Julia/Ada tooling.
. pull build artifacts from the collector (Rust), analyzer (Deno/ReScript), publisher (Deno), dashboard (Elixir), scraper (Julia), and Ada TUI.
. leverage BuildKit `--mount=type=bind` + the `tools/scripts/container-build.sh` helper so the local Eclipse clones of Svalinn, Cerro Torre, Selur, and Vörðr contribute verified binaries without hitting the public GitHub network.
. stage artifacts via `cerro-torre` containerfiles in a reproducible pipeline.
. expose ports 3001/3002/3003/4000 for internal mapping while `selur-compose` wires the services together through Svalinn/Vörðr.

`just container-build` (calls `tools/scripts/container-build.sh`) tags the resulting `.ctp` bundle as `ghcr.io/hyperpolymath/social-media-monitor:latest`, accepts `LOCAL_REPOS` (default `/var/mnt/eclipse/repos`), and keeps `docker buildx`/BuildKit as the canonical builder.

`compose.toml` now describes the `nuj-stack` service—based on the Chainguard bundle—and the SurrealDB/VerisimDB + Dragonfly/Redis + Prometheus/Grafana/Loki data fabric. `selur-compose` reads this file to start the verified containers; legacy Podman compose files have been archived under `docs/legacy/podman`.

Selur-compose governs lifecycle (`selur-compose up`, `down`, `scale`, `logs`, `config`) and obeys the verified container spec enforced by Svalinn. The `selur-compose` repository already documents 25+ commands, graph resolution, and integration with Svalinn, Vörðr, and Cerro Torre; our repo uses it as the orchestration plane.

== Data Fabric & Observability

* **SurrealDB + VerisimDB** – SurrealDB is the primary multi-model store for platform metadata, policy documents, and member guidance, while VerisimDB captures immutable, high-resolution time-series (policy snapshots, change events) with the same schema definitions.
* **Virtuoso RDF triple store** – PESTLE observatory knowledge graph, SPARQL endpoints, Linked Data for guidance provenance.
* **XTDB (Clojure)** – complete bitemporal audit trail that supports full history queries, essential for compliance reviews.
* **Dragonfly cache** – replaces Redis for high-throughput caching, job coordination, and pub/sub; retains Redis protocol for compatibility.
* **Redis job queues** – short-lived work items (scrape triggers, analyzer tasks) and Celery-like coordination.
* **Prometheus + Grafana + Loki** – metrics, dashboards, and log aggregation; every service exposes Prometheus metrics and structured JSON logs.

== Development Flow & CLI

[cols="1,1"]
|===
| Command | Purpose
| `just setup` | Configure dev dependencies, envvars, generate Chainguard stage definitions, run `guix pull` (hyperpolymath channel), and enable the `flake.nix` Nix fallback shell.
| `just dev` | Launch local dev mode via `selur-compose` connected to Svalinn/Vörd; exposes the dashboard on port 4000.
| `just test` | Runs per-service suites (Rust, ReScript, Elixir, Julia, Ada) plus integration checks.
| `just container-build` | Runs `tools/scripts/container-build.sh`, builds the Chainguard Containerfile with local Eclipse repos, and tags `ghcr.io/hyperpolymath/social-media-monitor:latest.ctp`.
| `selur-compose up` | Start the integrated stack (see `compose.toml`), managed by Svalinn policy gating and Vörðr runtime enforcement.
| `just logs <service>` | Tail structured logs via Loki/Grafana pipeline; reference `tools/scripts/dev-logs.sh` for specifics.
|===

The `tools/scripts/` directory now orchestrates container-aware dev servers (no Podman). Each script calls the new `selur-compose` CLI or the Chainguard build; the repo keeps the legacy Podman artifacts for reference only and does not invoke them in production.
